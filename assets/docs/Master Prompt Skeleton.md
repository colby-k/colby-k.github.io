## Prompt Master Skeleton

### Role
You are a **[role]** specializing in **[domain]**.

### Objective
Create **[deliverable]** for **[audience]** that achieves **[specific outcome]**.

### Context
- **Setting:** [meeting / report / email / analysis]
- **Time/Length:** [e.g., 30 minutes / one page / 200 words]
- **Priorities:** [what matters most]
- **Tone:** [professional / direct / executive-friendly]
- **Constraints/Environment:** [tools, systems, stakeholders, deadlines]

### Inputs
I will provide:
- **[Input 1]** (e.g., meeting notes, transcript, prior-quarter summaries)
- **[Input 2]** (e.g., template, policy, dataset)

Treat **[most recent / most authoritative source]** as the highest priority.

### Task
1) **Extract**: [key facts, themes, decisions, risks, action items]  
2) **Analyze**: [patterns, gaps, priorities, tradeoffs, dependencies]  
3) **Produce**: [the deliverable]  
4) **Include**: [follow-ups, open questions, prep items, assumptions as needed]

### Output Format Requirements
- **Structure:** [table / headings / bullets]
- **Must include fields:** [field 1], [field 2], [field 3]
- **Limits:** [max bullets / max sections / max length]
- **Style rules:** [e.g., “use short bullets,” “avoid jargon,” “one page only”]

### Quality Criteria
Make the output:
- **Specific** (not generic)
- **Actionable** (clear next steps)
- **Audience-ready** (appropriate tone and detail)
- **Aligned to the objective**

### Guardrails
- **Do not invent facts.** If unclear, write: **“Need to confirm.”**
- **If assumptions are required, label them clearly.**
- **Avoid filler.** Prioritize the highest-impact items first.

### Optional (if helpful)
- **Examples:** Provide 1–2 examples of “good” vs “too generic”
- **Checklist:** Include a short checklist to validate completeness
